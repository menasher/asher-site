## The brief

A Series-B SaaS shipped a Figma library and a production codebase that
disagreed about most things — colors, spacing, even some component names.
The team wanted them reconciled before launching the next major release.

## My role

Audited every token across both surfaces, drafted a 240-item ledger of
*conflict / source-of-truth / migration plan*, then paired with the
engineering team to ship the changes in three increments.

## Outcomes

- **47% fewer UI bugs** filed per sprint (8-week comparison)
- Design tokens now drive Tailwind config — single source of truth
- One Figma library, one CSS variable file, no more ad-hoc hex values
